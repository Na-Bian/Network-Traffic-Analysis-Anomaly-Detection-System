📖 1. 系統簡介

本系統是華中科技大學網路空間安全學院《程式設計綜合課程設計》課程的實踐成果，旨在將電腦網路、資料結構、演算法設計與 C++/Python 程式設計等理論知識綜合應用於貼近工程實際的網路安全分析場景。面對日益複雜的網路攻擊手段，對海量流量進行即時監控與智慧分析已成為保障網路穩定運行的關鍵。本系統緊扣實際需求，能夠高效處理真實的網路會話數據，自動建構網路拓撲模型，並整合流量統計、行為分析、異常檢測與結果視覺化等一系列功能，為網路管理員和安全分析師提供直觀、高效的一體化分析工具。

🔧 5. 核心功能詳解

5.1 流量排序

位於第一個標籤頁。用於快速發現網路中最活躍或最可疑的節點。

  • 總流量排序：統計每個 IP 節點的總出入流量，按流量降序排列。
  • HTTPS 流量排序：篩選使用HTTPS協定（傳輸層協定為 TCP 且目的連接埠為 443 ）進行通訊的會話，統計相關節點的 HTTPS 流量並降序排列。
  • 出流量佔比：篩選出流量佔總流量比例超過指定閾值（預設 80%）的節點，按總流量降序排列，並顯示具體佔比。

點擊“執行流量排序”後，結果將顯示在“資料表格”子頁。如您需要匯出詳細結果，可從“運行日誌”中複製對應文本。

5.2 路徑查找

指定來源 IP 和目的 IP，系統將基於所選策略計算最優路徑。

  • 最小壅塞路徑：以 資料量/持續時間 作為邊的壅塞度量，尋找總壅塞值最低的路徑。
  • 最小跳數路徑：尋找跳數最少的路徑，即最短路由。
  • 最小風險路徑：透過自訂代價函數（考慮連接埠安全性及連接埠掃描行為），尋找風險值最低的路徑。
  • 對比三種策略：勾選此核取方塊後，點擊按鈕將同時計算三種策略下的最優路徑，並在“路徑與詳情”子頁進行對比展示。

點擊“查找路徑”後，系統自動切換到“路徑與詳情”子頁，展示路徑的 IP 序列及對應的壅塞/風險值及其他詳細資訊。

5.3 異常檢測

標籤頁內包含三個子頁：連接埠掃描、DDoS 目標、星型結構。

📡 連接埠掃描檢測

如果一個節點對同一鄰居IP發起了大量不同目的連接埠的連接，且該節點的出流量佔總流量的比例超過閾值，則標記為可疑掃描者。預設出流量佔比閾值為80%，連接埠數閾值為20，您可以調整這些值以控制檢測靈敏度。

💥 DDoS 目標檢測

如果一個節點接收來自大量不同鄰居的連接請求且入流量超過閾值，則標記為可能的 DDoS 攻擊目標。預設鄰居數閾值為20，入流量閾值為1GB，您可以調整這些值以控制檢測靈敏度。

⭐ 星型結構查找

如果一個節點與大量葉子節點（只與該節點連接）通訊，則標記為星型結構中心。預設葉子節點數閾值為20，您可以調整該值以控制檢測靈敏度。

點擊各子頁的“檢測”按鈕後，結果將顯示在“資料表格”中，並自動生成對應的子圖，突出展示異常節點及其連接關係。

5.4 自訂規則

本功能允許使用者根據 IP 範圍、協定、連接埠、流量閾值自訂存取控制規則，並找出所有違反規則的通訊記錄。

  • 目標 IP：要監控的中心節點。
  • IP 範圍：可選擇 CIDR 格式（如 192.168.1.0/24）或起始-結束 IP 範圍。
  • 可選參數：協定號、來源連接埠、目的連接埠、最大流量閾值。
  • 規則類型：“拒絕”表示目標 IP 禁止與範圍內 IP 通訊，且禁止使用指定協定/連接埠；“允許”表示目標 IP 只能與範圍內 IP 通訊，且只能使用指定協定/連接埠。

點擊“檢測違規記錄”後，“資料表格”子頁將展示所有違反規則的通訊記錄，包括IP位址、協定、連接埠、違規原因等，如您需要匯出詳細結果，可從“運行日誌”中複製對應文本。

5.5 子圖視覺化

輸入任意 IP 位址，點擊“生成子圖”，系統將找出該節點所在的連通分量，並利用 PyVis 引擎渲染成交互式網路拓撲圖，顯示在右側畫布中。您可以：

  • 拖拽節點、滾輪縮放。
  • 懸停節點查看 IP 位址，懸停邊查看流量資訊。
  • 通過功能表“匯出”儲存當前子圖為 JSON 或 HTML 檔案。

💾 6. 匯出功能

通過功能表 檔案 → 匯出，您可以將分析結果儲存為多種格式：

匯出項格式說明
PCAP 轉換的 CSV.csv若原始資料為 PCAP 且已完成轉換，可匯出為 CSV 檔案供外部使用。
全網拓撲.json / .html將整個網路拓撲圖匯出為 JSON（資料）或 HTML（互動式視覺化）檔案。

📌 小提示：匯出的 HTML 檔案可直接用瀏覽器開啟，預設使用線上資源進行解析。

⚙️ 7. 設定與個性化

7.1 執行緒數

功能表 設定 → 執行緒數 可調整後台分析使用的CPU核心數。預設為4。

7.2 語言切換

點擊功能表 設定 → 語言 / Language 可選擇簡體中文、繁體中文或英語。部分介面需要重啟程式才能生效。

7.3 主題自適應

系統自動跟隨 Windows 系統淺色/深色模式，調整介面背景、文字顏色及圖表配色。您可以通過修改系統主題來改變程式外觀。

🔍 8. 僅檢視模式

當您直接開啟一個 .json 或 .html 檔案時，系統自動進入僅檢視模式：

  • 所有資料分析功能不可用。
  • 僅允許瀏覽已有的圖結構、查看節點/邊屬性。
  • 您仍可執行檔案匯出操作。

此模式適用於分享分析結果或在沒有原始資料時回顧之前儲存的拓撲。

❓ 9. 常見問題（FAQ）

Q: 載入 PCAP 檔案後很長時間沒反應？
A: 大型 PCAP 檔案轉換可能耗時較長，請耐心等待。
Q: 生成子圖時顯示“圖中無任何節點”？
A: 可能原因：① 輸入的 IP 位址不在當前拓撲中；② 沒有與該 IP 通訊的其他節點；③ CSV 檔案中未包含有效 IP。請先驗證該節點是否存在。
Q: 大型網路拓撲圖載入緩慢？
A: 受到 PyVis 與 Qt WebEngine 效能限制，建議在載入資料後輸入特定 IP 以縮小子圖範圍，避免渲染全圖。
Q: 自訂規則查找結果為空？
A: 請檢查目標 IP 是否正確，以及 IP 範圍、協定、連接埠等條件是否過於嚴格。您可以逐步放寬條件測試。
Q: “最小風險路徑”中網路通訊的風險程度是如何定義的？
A: 風險程度基於邊的“風險代價”進行評估。每條邊的風險代價由兩部分構成：
  • 基礎安全係數：若通訊涉及安全協定（如 HTTPS、SSH 等），係數為 1.0；否則為 10.0。
  • 連接埠掃描懲罰係數：根據通訊中出現的不同連接埠對數量進行計算，連接埠對數量越多，懲罰係數越高（最大不超過 3.0）。
最終邊的風險代價 = 基礎安全係數 × 連接埠掃描懲罰係數。系統將尋找從來源到目的的所有路徑中，總風險代價最小的路徑作為“最小風險路徑”。該設計能夠有效識別並規避潛在的連接埠掃描行為和低安全性通訊鏈路。

📋 10. 附錄：資料格式參考

標準輸入 CSV 檔案必須包含以下欄位，編碼為 UTF-8：

Source,Destination,Protocol,SrcPort,DstPort,DataSize,Duration
192.168.1.10,10.0.0.22,6,54321,443,1200,0.5
192.168.1.15,10.0.0.22,6,12345,80,800,0.3
...

欄位說明

  • Source / Destination：點分十進位 IPv4 位址。
  • Protocol：IP 協定號（例如 6=TCP, 17=UDP, 1=ICMP）。
  • SrcPort / DstPort：傳輸層連接埠（若協定無連接埠概念則留空）。
  • DataSize：會話總位元組數。
  • Duration：會話持續時間（秒）。
📌 小提示：您可以使用 Wireshark 的“會話統計”功能匯出 CSV 資料，或直接使用原始 .pcap 檔案，系統會自動完成流聚合與預處理。
