## **📖 1\. 系统简介**

本系统是华中科技大学网络空间安全学院《程序设计综合课程设计》课程的实践成果，旨在将计算机网络、数据结构、算法设计与 C++/Python 编程等理论知识综合应用于贴近工程实际的网络安全分析场景。面对日益复杂的网络攻击手段，对海量流量进行实时监控与智能分析已成为保障网络稳定运行的关键。本系统紧扣实际需求，能够高效处理真实的网络会话数据，自动构建网络拓扑模型，并集成流量统计、行为分析、异常检测与结果可视化等一系列功能，为网络管理员和安全分析师提供直观、高效的一体化分析工具。

**✨ 核心能力速览**

* **📁 多格式数据支持**：读取 PCAP / CSV / JSON / HTML 数据  
* **📈 流量统计**：构建网络拓扑图，支持基于特定条件的筛选与流量统计分析  
* **🔍 路径分析**：最小拥塞、最小跳数、最小风险、多策略对比  
* **🚨 异常检测**：端口扫描、DDoS 目标、星型结构  
* **⚙️ 自定义安全规则**：IP 范围 / 协议 / 端口规则，查找违规通信记录  
* **🖼️ 交互式可视化**：全网拓扑、子图、路径的可视化展示，支持导出存档与分享

**🛠️ 开发环境与编程语言**

* **C++ 核心后端**：使用 CLion 2025.3.3 开发，采用 C++23 标准，最终二进制文件由 Microsoft Visual Studio 2026 (MSVC v143) 编译生成。  
* **Python 前端与工具**：基于 Python 3.14，在 PyCharm 2025.3.3 中开发，负责 PCAP 解析、可视化渲染及 GUI 界面。  
* **图形界面**：由 PyQt6 构建的现代化界面，结合 Qt WebEngine 展示交互式网络拓扑。

### **4.1 支持的文件类型**

| 格式 | 说明 |
| :---- | :---- |
| .csv | 预处理好的标准网络会话数据，包含源IP、目的IP、协议、端口、流量、持续时间等字段。 |
| .pcap | 通过 Wireshark 等工具捕获的原始网络字节流抓包文件。 |
| .json | 先前导出的网络拓扑数据（节点、边及属性），可重新加载并查看。 |
| .html | 先前导出的交互式拓扑可视化文件，可直接在浏览器或本系统中打开。 |

### **4.2 加载数据**

点击工具栏的“浏览”按钮，或通过菜单 文件 → 打开数据文件，选择目标文件即可加载。

* 若选择 **PCAP 文件**，程序将自动调用后台解析模块，将其转换为 CSV 格式并加载。请注意，大型文件加载速度可能较慢。  
* 若选择 **JSON 或 HTML 文件**，程序将进入\*“仅查看模式”\*。您可以浏览已有的图结构，或继续导出为其他格式，但无法执行新的分析操作。

**📌 小提示**：系统运行过程中产生的临时文件默认存放在系统临时目录，您可以定期清理以释放空间。

## **🔧 5\. 核心功能详解**

### **5.1 流量排序**

位于第一个选项卡。用于快速发现网络中最活跃或最可疑的节点。

* **总流量排序**：统计每个 IP 节点的总出入流量，按流量降序排列。  
* **HTTPS 流量排序**：筛选使用HTTPS协议（传输层协议为 TCP 且目的端口为 443 ）进行通信的会话，统计相关节点的 HTTPS 流量并降序排列。  
* **出流量占比**：筛选出流量占总流量比例超过指定阈值（默认 80%）的节点，按总流量降序排列，并显示具体占比。

点击“执行流量排序”后，结果将显示在\*\*“数据表格”\*\*子页。如您需要导出详细结果，可从“运行日志”中复制对应文本。

### **5.2 路径查找**

指定源 IP 和目的 IP，系统将基于所选策略计算最优路径。

* **最小拥塞路径**：以 数据量/持续时间 作为边的拥塞度量，寻找总拥塞值最低的路径。  
* **最小跳数路径**：寻找跳数最少的路径，即最短路由。  
* **最小风险路径**：通过自定义代价函数（考虑端口安全性及端口扫描行为），寻找风险值最低的路径。  
* **对比三种策略**：勾选此复选框后，点击按钮将同时计算三种策略下的最优路径，并在“路径与详情”子页进行对比展示。

点击“查找路径”后，系统自动切换到\*\*“路径与详情”\*\*子页，展示路径的 IP 序列及对应的拥塞/风险值及其他详细信息。

### **5.3 异常检测**

选项卡内包含三个子页：端口扫描、DDoS 目标、星型结构。

#### **📡 端口扫描检测**

如果一个节点对同一邻居IP发起了大量不同目的端口的连接，且该节点的出流量占总流量的比例超过阈值，则标记为可疑扫描者。默认出流量占比阈值为 80%，端口数阈值为 20，您可以调整这些值以控制检测灵敏度。

#### **💥 DDoS 目标检测**

如果一个节点接收来自大量不同邻居的连接请求且入流量超过阈值，则标记为可能的 DDoS 攻击目标。默认邻居数阈值为 20，入流量阈值为 1GB，您可以调整这些值以控制检测灵敏度。

#### **⭐ 星型结构查找**

如果一个节点与大量叶子节点（只与该节点连接）通信，则标记为星型结构中心。默认叶子节点数阈值为 20，您可以调整该值以控制检测灵敏度。

点击各子页的“检测”按钮后，结果将显示在\*\*“数据表格”\*\*中，并自动生成对应的子图，突出展示异常节点及其连接关系。

### **5.4 自定义规则**

本功能允许用户根据 IP 范围、协议、端口、流量阈值自定义访问控制规则，并找出所有违反规则的通信记录。

* **目标 IP**：要监控的中心节点。  
* **IP 范围**：可选择 CIDR 格式（如 192.168.1.0/24）或起始-结束 IP 范围。  
* **可选参数**：协议号、源端口、目的端口、最大流量阈值。  
* **规则类型**：“拒绝”表示目标 IP 禁止与范围内 IP 通信，且禁止使用指定协议/端口；“允许”表示目标 IP 只能与范围内 IP 通信，且只能使用指定协议/端口。

点击“检测违规记录”后，\*\*“数据表格”\*\*子页将展示所有违反规则的通信记录，包括IP地址、协议、端口、违规原因等，如您需要导出详细结果，可从“运行日志”中复制对应文本。

### **5.5 子图可视化**

输入任意 IP 地址，点击“生成子图”，系统将找出该节点所在的连通分量，并利用 PyVis 引擎渲染成交互式网络拓扑图，显示在右侧画布中。您可以：

* 拖拽节点、滚轮缩放。  
* 悬停节点查看 IP 地址，悬停边查看流量信息。  
* 通过菜单“导出”保存当前子图为 JSON 或 HTML 文件。

## **❓ 9\. 常见问题（FAQ）**

**Q: 加载 PCAP 文件后很长时间没反应？**

**A:** 大型 PCAP 文件转换可能耗时较长，请耐心等待。

**Q: 生成子图时显示“图中无任何节点”？**

**A:** 可能原因：① 输入的 IP 地址不在当前拓扑中；② 没有与该 IP 通信的其他节点；③ CSV 文件中未包含有效 IP。请先验证该节点是否存在。

**Q: 大型网络拓扑图加载缓慢？**

**A:** 受到 PyVis 与 Qt WebEngine 性能限制，建议在加载数据后输入特定 IP 以缩小子图范围，避免渲染全图。

**Q: 自定义规则查找结果为空？**

**A:** 请检查目标 IP 是否正确，以及 IP 范围、协议、端口等条件是否过于严格。您可以逐步放宽条件测试。

**Q: “最小风险路径”中网络通信的风险程度是如何定义的？**

**A:** 风险程度基于边的“风险代价”进行评估。每条边的风险代价由两部分构成：

• **基础安全系数**：若通信涉及安全协议（如 HTTPS、SSH 等），系数为 1.0；否则为 10.0。

• **端口扫描惩罚系数**：根据通信中出现的不同端口对数量进行计算，端口对数量越多，惩罚系数越高（最大不超过 3.0）。

最终边的风险代价 \= 基础安全系数 × 端口扫描惩罚系数。系统将寻找从源到目的的所有路径中，总风险代价最小的路径作为“最小风险路径”。该设计能够有效识别并规避潜在的端口扫描行为和低安全性通信链路。

## **📋 10\. 附录：数据格式参考**

标准输入 CSV 文件必须包含以下列，编码为 UTF-8：

Source,Destination,Protocol,SrcPort,DstPort,DataSize,Duration  
192.168.1.10,10.0.0.22,6,54321,443,1200,0.5  
192.168.1.15,10.0.0.22,6,12345,80,800,0.3  
...

**字段说明**

* Source / Destination：点分十进制 IPv4 地址。  
* Protocol：IP 协议号（例如 6=TCP, 17=UDP, 1=ICMP）。  
* SrcPort / DstPort：传输层端口（若协议无端口概念则留空）。  
* DataSize：会话总字节数。  
* Duration：会话持续时间（秒）。

**📌 小提示**：您可以使用 Wireshark 的“会话统计”功能导出 CSV 数据，或直接使用原始 .pcap 文件，系统会自动完成流聚合与预处理。
